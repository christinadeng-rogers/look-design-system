# HTML: Write Clean Code

Write clean and easy-to-understand HTML code is the foundation of a beautiful website.
It also plays an important role in SEO. <br/> Use validation tool to validate your html code: <a>https://validator.w3.org/</a>.

Here are some principles for writting clean HTML code:

<div className="subheading">2. Write Standards-Compliant Markup</div>
<div className="link-list">
  <a
    className="link-item"
    href="https://chiamakaikeanyi.dev/writing-standards-compliant-html/"
    target="_blank"
  >
    <span>
      <div className="section-1">
        <h6>
          Accessibility cannot be achieved without quality and semantically
          valid HTML
        </h6>
        <ul>
          <li>Make sure to open and close and nest elements properly.</li>
          <li>
            Indent the code and nested components to show hierarchy. Indented
            code is easier to read and maintain. (Install Prettier in VSC will
            save you lots of time)
          </li>
          <li>
            Separation of concerns: create separate HTML, CSS and JS files, it
            will make code reusable and easier to read.
          </li>
          <li>
            Be consistent: dash/underscore for ids and classes; single/double
            quote argument; indent spaces.
          </li>
          <li>
            Use lowercase for elements, attributes and values. leave typography
            to the CSS.
          </li>
          <li>
            Avoid Too Much Commenting: "Don't comment bad code. Rewrite it."
            Continually Refactor Code!
          </li>
        </ul>
      </div>
    </span>
  </a>
</div>

<div className="subheading">3. Make Use of Semantic Elements</div>
<div className="link-list">
  <a
    className="link-item"
    href="https://altcampus.school/posts/writing-semantically-correct-and-clean-html"
    target="_blank"
  >
    <span>
      <div className="section-1">
        <li>
          Alt text on image: provides a descriptive alternate text for images.
          It helps screen readers to understand contents better. Decorative
          images should have empty alt.
        </li>
        <li>
          Naming Convention: use meaningful id and class name that's related to
          the content (not the presentation). They should be short, descriptive
          and represent only one concept.
        </li>
        <li>Logical Sequence: useful to screen readers.</li>
        <li>
          Use elements appropriately: avoid writing many <code>div</code> in
          place of meaningul semantic HTML elements.
          <ul>
            <li>
              Avoid using
              <code>&lt;b&gt;</code>
              <code>&lt;u&gt;</code>
              <code>&lt;i&gt;</code>
              <code>&lt;s&gt;</code>
            </li>
            <li>
              Use
              <code>&lt;strong&gt;</code>
              <code>&lt;ins&gt;</code>
              <code>&lt;em&gt;</code>
              <code>&lt;del&gt;</code>
              instead, since they have meaning for screen readers.
            </li>
          </ul>
        </li>
      </div>
    </span>
  </a>
</div>
