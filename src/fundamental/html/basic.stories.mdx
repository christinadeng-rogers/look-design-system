      <h5>Shadow DOM vs. Virtual DOM</h5>
      <p>Shadow DOM</p>
      <ul>
        <li>
          Isolates the DOM so the DOM of the component is a separate element
          which will not appear in a global DOM.
        </li>
        <li>
          Styles created inside the single Shadow DOM element are isolated and
          stays in the scope of that Shadow DOM.
        </li>
        <li>It can affect the performance of the application.</li>
      </ul>
      <p>Virtual DOM</p>
      <ul>
        <li>
          Browser knows which elements were changed and can update only those
          part of the application to avoid re-rendering all the DOM.
        </li>
        <li>Improve the performance of the UI libraries.</li>
        <li>
          The biggest issue that Virtual DOM solves is the performance
          improvement on DOM manipulation.
        </li>
      </ul>
      <p>
        Common: they both help with performance issue. And both create a
        separate instance of the Document Object Model.
      </p>
      <p>Main differences</p>
